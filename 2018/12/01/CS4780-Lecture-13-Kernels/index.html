CS 4780 Lecture 13: Kernels

Posted on 2018-12-01 | In CS 4780 Study Notes

Why Kernels

Linear classifiers are great. But sometimes, even though there is a clear pattern in the input
data, it cannot be linearly seperated. Here’s an example of non linearly seperable data:

In this specific case, we could add another dimension that is related to existing dimensions
(e.g. euclidean distance to the origin) to make the data linearly seperable without changing
existing features. This is the idea of Kernel.

Feature Expansion

For a data vector $x \in \mathbf{R}^{d} $, we apply a transformation function $x \rightarrow \phi(x)$
where $\phi(x) \in \mathbf{R}^{D}$. Usually $D >> d$ because we add dimensions that capture non-linear
interactions among the original features.

Advantage: The approach is simple and the new vector stays convex and well behaved.

Disadvantage: The dimension could be very high.

Quiz

Q: At what regime do you want to add more features? High variance or high bias?
A: High bias.

Kernel Trick

The kernel trick is based on the observation: if we are doing gradient descent with some standard loss functions (for example, square loss), the gradient is actually a linear combination of input training vectors (this can be proved by induction using the fact that our kernel vector stays convex). In this case, we don’t even need to compute a single vector $\phi(x)$ or the full gradient vector $w$. Hence we get around the dilemma that dimention of kernel vector could go very high.

We could represent the gradient vector (after t computations) as follows:
$$w_{t} = \sum_{i=1}^{n}\alpha_{i}^{t}x_{i}$$

Suppose we are using square loss function, then $$l(w) = \sum_{i=1}^{n}(\sum_{j=1}^{n}\alpha_{j}x_{j}^{T}x_{i}-y_{i})^2$$
$$h(x_{t}) = w^Tx_{t} = \sum_{j=1}^{n}\alpha_{j}x_{j}^Tx_{t}$$

Inner Product Computation

Suppose we have two kernel vector $\phi(x)$ and $\phi(z)$, the inner product of these two is:
$$\phi(x)^{T}\cdot\phi(z) = 1\cdot 1 + x_{1}z_{1} + x_{2}z_{2} + … + x_{1}x_{2}z_{1}z_{2} + … + x_{1}…x_{d}z_{1}…z_{d} = \prod_{k=1}^{d}(1+x_{k}z_{k})$$
In this way, we reduced the computation of sum of $2^d$ terms to the computation of the product of $d$ terms (i.e. the complexity reduced from $O(2^2)$ to $O(d)$).
The kernel function is defined as $k(x_{i}, x_{j}) = \phi(x_{i})^{T}\cdot\phi(x_{j})$.
With $n$ input vectors, we have a kernel matrix defined as $K_{ij} = k(x_{i}, x_{j})$.
Then $h(x_{t}) = \sum_{j=1}^{n}\alpha_{j}k(x_{j}, x_{t})$

Choice of Kernel Functions

A matrix can be a kernel matrix if and only if it is positive semi-definite.

A matrix $A \in \mathbf{R}^{N \times N}$ is p.s.d if and only if $\forall q \in \mathbf{R}^{N}, q^{T}Aq \geq 0$
